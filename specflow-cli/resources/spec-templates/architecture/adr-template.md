# ADR 模板

本文件是架构决策记录（ADR）的填写模板。复制本文件为 `adr-<编号>-<标题>.md` 后填写。

---

# ADR-NNN: [决策标题]

> 一句话概括本决策做了什么选择。

## 元信息

| 字段 | 值 |
|------|------|
| 编号 | ADR-NNN |
| 日期 | YYYY-MM-DD |
| 状态 | 提案 / 已接受 / 已废弃 / 已被替代 |

## 上下文

<!-- 引导说明：描述做出决策时面临的问题与约束。回答"为什么需要做这个决策"。 -->

（由团队填写：说明决策的触发场景、当前痛点、业务或技术约束、相关利益方。这部分是未来回看时最有价值的内容，务必具体。）

## 决策

<!-- 引导说明：描述最终选择了什么。简洁、明确、可执行。 -->

（由团队填写：写出选择方案及其关键参数。例如"采用 PostgreSQL 作为主数据库，使用 Prisma 作为 ORM"。）

## 后果

<!-- 引导说明：描述这个决策带来的正面与负面影响。诚实记录代价。 -->

（由团队填写：分"正面后果"与"负面后果 / 代价"两部分。包括对开发流程、性能、运维、团队技能要求的影响。）

## 替代方案

<!-- 引导说明：描述考虑过但未选择的方案，以及为什么放弃。这章节防止未来重复纠结同一问题。 -->

（由团队填写：列出每个候选方案及其被放弃的原因。例如：

- **方案 A**：xxx。放弃原因：xxx。
- **方案 B**：xxx。放弃原因：xxx。
）

## 状态变更记录

<!-- 引导说明：仅在 ADR 状态发生变更时填写。 -->

（如本 ADR 被废弃或替代，在此记录：YYYY-MM-DD，状态变更为 xxx，参见 ADR-NNN。）
