## 流水线阶段

（由团队填写：定义 CI/CD 流水线的阶段划分。包括：

标准阶段示例：

1. **Checkout** — 拉取代码
2. **Install** — 安装依赖
3. **Lint** — 代码风格检查
4. **Build** — 编译 / 构建
5. **Test** — 单元测试 + 集成测试
6. **Scan** — 安全扫描 / 依赖检查
7. **Package** — 打包 / 构建镜像
8. **Deploy (staging)** — 部署到 staging
9. **E2E Test** — 端到端测试
10. **Deploy (prod)** — 部署到生产（需审批）

包括：
- 每个阶段的触发条件
- 阶段间的依赖关系
- 失败后的处理策略（停止 / 继续 / 通知）
- 哪些阶段在 PR 时运行、哪些在合并后运行
- 代码示例：流水线配置文件
）

## 常见错误

（由团队填写：列出团队在 CI/CD 上犯过的错误，例如：

- CI 流水线太慢（>30 分钟），开发者开始跳过 CI
- 缓存 key 配置错误，缓存从未命中
- CI 中的 secret 泄露在日志中
- 流水线配置复制粘贴，维护时遗漏同步
- 构建产物无版本标签，无法追溯哪个 commit 对应哪个产物
- CI 失败无通知，问题积累到发版时才发现
- E2E 测试在 CI 中 flaky，团队习惯性重试而非修复
- 部署阶段无审批，直接部署到生产
- CI 配置修改后未测试，下次合并时才发现流水线挂了
）
