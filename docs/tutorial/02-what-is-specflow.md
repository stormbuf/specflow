# Specflow 是什么

Specflow 是一个 **Spec 驱动的变更生命周期管理工具**。它由 Go CLI、平台插件和 auto-trigger skills 三部分组成，安装在任意项目里，让 AI 编码助手在变更全生命周期中自动获得正确的上下文、阶段感知和跨会话记忆。

它的核心能力可以概括为一句话：**让 AI 在正确的时机，拿到正确的上下文，按照正确的流程，做正确的事**。

## 它解决了什么问题

| 痛点 | 没有 Specflow | 有 Specflow |
|------|--------------|-------------|
| AI 不知道项目规范 | 每次对话都要手动告诉 AI 编码规范 | spec 库自动注入，AI 编码前先读规范 |
| AI 不知道当前进度 | 新会话从零开始，不知道上次做到哪 | workflow.md 面包屑每轮自动注入当前阶段 |
| AI 派发子 agent 缺上下文 | 子 agent 拿到什么全靠编排器"自觉" | jsonl manifest 声明上下文文件与顺序，插件自动注入 |
| 需求没有可追溯的验收标准 | "感觉对了"就算完成 | prd.md 用 EARS + Gherkin 定义验收标准，check skill 逐条验证 |
| 踩过的坑下次还踩 | 每次从零开始，经验不沉淀 | update-spec skill 把经验写入 spec 库，下次自动读取 |
| 跨会话遗忘 | 新窗口不知道之前的对话和决策 | journal 记录会话日志，mem 检索历史对话 |

## 它不做什么

- **不是代码生成器** — 它不直接写代码，而是为 AI 编码助手提供结构化的工作流和上下文
- **不是 IDE 插件** — 它是一个独立的 CLI 工具 + 平台插件，与 AI 编码助手配合使用
- **不替代你的判断** — 它提供框架和约束，但需求决策、架构选择仍然由你做主
- **不强制使用** — 简单对话和小改动可以跳过任务流程，直接 inline 处理
